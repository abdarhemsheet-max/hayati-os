# نظام حياتي 🌟

نظام شخصي متكامل لإدارة الحياة (Life OS) — لعبدالرحيم أحمد شيتة.

**محلي 100%** — لا خدمات سحابية، لا حسابات، لا إنترنت بعد التثبيت. بياناتك كلها في ملف واحد على جهازك.

**التقنيات:** Next.js 14 (App Router) · TypeScript · Tailwind CSS · **SQLite + Prisma ORM** — بتصميم Glassmorphism داكن كامل الدعم للعربية (RTL).

## هيكلية الملفات 🗂️

الكود مقسّم بوضوح حسب الغرض لتسهيل التعديل:

```
src/
├── app/                    ← التوجيه فقط (يفرضه Next.js)
│   ├── api/                    نقاط النهاية (Backend endpoints) — route.ts
│   └── (dashboard)/             صفحات الواجهة (Frontend pages) — page.tsx
├── backend/                ← منطق الباك اند بالكامل
│   ├── prisma.ts               عميل قاعدة البيانات
│   ├── resources.ts            تعريف كل موارد الـ CRUD
│   ├── validate.ts             التحقق من صحة المدخلات
│   ├── uploads.ts               إعدادات رفع الملفات
│   └── learningSync.ts         مزامنة عدادات الكورسات
├── frontend/                ← منطق ومكوّنات الواجهة بالكامل
│   ├── api.ts                   عميل الاتصال بالـ API + الكاش
│   ├── youtube.ts               أدوات صور يوتيوب المصغرة
│   └── components/               كل مكوّنات الواجهة (Sidebar, GlassCard, ...)
└── shared/                  ← يُستخدم من الطرفين
    ├── types.ts                 أنواع البيانات المشتركة
    └── utils.ts                 دوال التنسيق والتواريخ
```

> ملاحظة تقنية: Next.js (App Router) يُلزم بقاء نقاط الـ API وصفحات الواجهة داخل `src/app/` كي يعمل التوجيه — لذا الفصل الفعلي بين الباك اند والفرونت اند تم على مستوى **المنطق البرمجي** (`backend/` و`frontend/`)، بينما `app/api/` و`app/(dashboard)/` منظّمتان أصلاً كل في مجلدها الخاص.

## الأداء وسرعة التنقل ⚡

النظام يعمل في **وضع الإنتاج** (كل الصفحات مبنية مسبقاً) — التنقل بين الأقسام فوري (~65ms مقابل ثوانٍ في وضع التطوير). إضافة إلى:
- **كاش ذكي**: العودة لقسم زرته تعرض بياناته فوراً من الذاكرة ثم تُحدَّث بصمت.
- **تحديثات متفائلة (Optimistic UI)**: إنجاز المهام والعادات والسحب يظهر لحظياً قبل رد الخادم.
- **Lazy Loading**: مكتبة السحب والإفلات تُحمّل فقط عند فتح تبويب «كافة المهام».

## التشغيل

### الطريقة الأسهل ⭐
انقر نقراً مزدوجاً على **`START.bat`** — يثبت الحزم، ينشئ قاعدة البيانات، **يبني نسخة الإنتاج** (أول مرة فقط)، ويفتح المتصفح على `http://localhost:4400`.

> **بعد أي تعديل على الكود:** شغّل **`UPDATE.bat`** مرة واحدة ثم `START.bat`. (تشغيل `npm run dev` يمسح نسخة الإنتاج، لذا START.bat يعيد بناءها تلقائياً عند الحاجة.)

### يدوياً
```bash
npm install          # يولّد عميل Prisma تلقائياً
npx prisma db push   # ينشئ قاعدة البيانات المحلية (prisma/dev.db)
npm run build        # يبني نسخة الإنتاج السريعة
npm run start        # ثم افتح http://localhost:4400
```

## الحماية من الأخطاء (No Red Screens)

- **تحقق صارم** من كل المدخلات في طبقة API ([src/lib/validate.ts](src/lib/validate.ts)) برسائل عربية واضحة.
- **معاملات ذرّية** (`$transaction`) للعمليات المالية: إنشاء الحركة وتحديث رصيد المحفظة ينجحان معاً أو يفشلان معاً.
- أي خطأ (تحقق/قاعدة بيانات/شبكة) يتحول إلى **تنبيه لطيف (Toast)** بدل انهيار الواجهة ([src/lib/api.ts](src/lib/api.ts)).
- علاقات Prisma كلها بـ `onDelete` صريح (Cascade / SetNull) — لا سجلات يتيمة ولا أخطاء حذف.
- قيود على مستوى القاعدة: `unique` لسجلات الأيام تمنع التكرار، والتقارير لا تقبل إلا حالة `archived`.

## ملاحظات

- **النسخ الاحتياطي:** انسخ ملف `prisma/dev.db` (قاعدة البيانات) ومجلد `uploads/` (المستندات) — هذا كل شيء.
- **تصدير PDF للتقارير:** زر «تصدير PDF» ثم «حفظ كـ PDF» من نافذة الطباعة — دعم كامل للعربية.
- **العملة:** غيّرها من `src/lib/utils.ts` (المتغير `CURRENCY`).
- **استعراض قاعدة البيانات:** `npm run db:studio` يفتح Prisma Studio.
